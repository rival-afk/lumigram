# PROJECT_SPEC.md – Lumigram (с правками)

## 1. Краткая суть

**Lumigram** — форк exteraGram (Telegram for Android), цель которого — **открытая платформа для плагинов на Python и C++**.
Плагины могут менять всё: от цвета кнопок до перехвата системных вызовов, от визуальных тем до шпионажа за удалёнными сообщениями. Главные принципы: **пользователь контролирует каждый бит доступа**, а система плагинов **отказоустойчива** и не крашит клиент.

Целевая аудитория — разработчики и продвинутые пользователи, желающие выжать максимум из Telegram, в том числе в условиях цензуры (блокировок РКН и т.п.) – за счёт встроенных механизмов обхода (Tor, VPN внутри плагинов).

---

## 2. Основные принципы (не нарушать)

1. **Полная совместимость с плагинами exteraGram** (файлы `.plugin`, API hook-ов, Chaquopy).
   Плагины Lumigram не обязаны работать в exteraGram, но обратная совместимость обязательна.

2. **Безопасность по умолчанию** – новый плагин не имеет прав ничего делать. Права выдаются пользователем в явном виде.

3. **Отказоустойчивость** – падение плагина не убивает клиент. Пользователь получает красивое сообщение (bulletin) с кодом ошибки и возможностью скопировать лог.

4. **Модульное ядро** – система плагинов, логгер, менеджер разрешений, SUDO-прослойка, UI – всё слабо связано. Можно менять части без переписывания всего.

5. **Поддержка Python и C++ плагинов**.
   - **Python** – полноценный интерпретатор с возможностью установки **любых внешних библиотек** (не только предустановленных, как в exteraGram). Плагин может иметь собственное виртуальное окружение с `pip` (разрешено только в песочнице).
   - **C++** – компилируемые плагины (`.so`) для максимальной производительности, прямого доступа к низкоуровневым API (TDLib, сокеты, root-команды). С поддержкой **root-прав** (через SUDO-менеджер) C++ плагины могут, например, обучать нейросети на GPU телефона или поднимать Tor-туннель для обхода блокировок.

## 3. Архитектура (высокоуровнево)

```
┌─────────────────────────────────────────┐
│         Lumigram (Java/Kotlin)          │
│  ┌────────────────────────────────────┐ │
│  │         Plugin Manager             │ │
│  │  – загрузка .plugin (Python/C++)   │ │
│  │  – безопасный режим                │ │
│  │  – управление жизненным циклом     │ │
│  └────────────────────────────────────┘ │
│  ┌────────────────────────────────────┐ │
│  │         Permission System          │ │
│  │  – запрос разрешений               │ │
│  │  – маппинг на Android/Telegram API │ │
│  └────────────────────────────────────┘ │
│  ┌────────────────────────────────────┐ │
│  │         Logging & Crash Handler    │ │
│  │  – структурированные логи          │ │
│  │  – безопасный режим при краше      │ │
│  └────────────────────────────────────┘ │
│  ┌────────────────────────────────────┐ │
│  │         SUDO / Shizuku Bridge      │ │
│  │  – выполнение привилегированных    │ │
│  │    команд (root / Shizuku)         │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
            │ JNI │
┌─────────────────────────────────────────┐
│      C++ Core (liblumigram.so)          │
│  – высокопроизводительные хуки          │

└─────────────────────────────────────────┘
```

**Коммуникация**: Java-часть вызывает C++ через JNI. Python-код выполняется внутри C++ (встроенный интерпретатор с возможностью создания виртуального окружения). C++ плагины компилируются как `.so` и динамически подгружаются через `dlopen`.

## 4. Система плагинов (детали)

### 4.1. Формат плагина
`.plugin` – zip-архив со структурой:

```
plugin_id.plugin/
├── manifest.json          # метаданные и список файлов
├── main.py                # главный скрипт (для Python)
├── plugin.so              # главная библиотека (для C++)
├── libs/                  # дополнительные модули (Python)
│   ├── helper.py
│   └── ...
├── assets/                # ресурсы (изображения, шрифты)
└── requirements.txt       # опционально: список pip-пакетов
```

**Пример `manifest.json`**:

```json
{
  "id": "spy_keeper",
  "name": "Spy Keeper",
  "version": "1.0.0",
  "author": "@username",
  "description": "Сохраняет удалённые сообщения",
  "min_sdk": 26,
  "min_lumigram_version": "1.0.0",
  "type": "python",            // "python" или "cpp"
  "entrypoint": "main.py",     // для Python
  "cpp_library": "plugin.so",  // для C++
  "permissions": [
    "telegram.read_messages",
    "lumigram.spy_mode",
    "lumigram.secret_storage"
  ],
  "optional_permissions": [
    "sudo.root_command"
  ],
  "requirements": "requirements.txt", // опционально
  "modules": [
    "libs/helper.py"
  ]
}
```

### 4.2. Модульность Python-плагинов
- `main.py` – точка входа. В нём определяется класс, наследующий `BasePlugin` (как в exteraGram).
- Дополнительные модули (`.py`) указываются в `manifest.json` → `modules`. Они копируются в рабочую директорию плагина и могут быть импортированы из `main.py` стандартным `import`.
- Плагин может иметь собственное виртуальное окружение: если присутствует `requirements.txt`, Lumigram создаёт изолированную среду в `data/plugins/<id>/venv` и устанавливает пакеты через `pip`. Это позволяет использовать любые сторонние библиотеки (например, `torch`, `transformers`, `stem` для Tor, `requests` и т.д.), недоступные в стандартной поставке exteraGram.

### 4.3. C++ плагины с root-доступом
- C++ плагины компилируются под целевую архитектуру (arm64, armeabi-v7a) и поставляются в виде `plugin.so`.
- Имеют полный доступ к C++ API Lumigram (TDLib, сокеты, JNI-вызовы Java-методов).
- При запросе разрешения `sudo.root_command` плагин может выполнять любые системные команды от root (через SUDO-менеджер). Это позволяет:
  - Запускать Tor (бинарник из ассетов) с привилегиями для создания туннеля.
  - Использовать GPU через OpenCL или Vulkan для обучения лёгких нейросетей (например, распознавание лиц, голоса).
  - Перехватывать системные события (скриншоты, звонки, смс) на низком уровне.
  - Модифицировать ядро Android (если устройство разблокировано).

### 4.4. Жизненный цикл плагина
1. **Установка** – копирование `.plugin` в `/data/data/.../plugins/<id>/`.
2. **Распаковка и проверка** – валидация `manifest.json`, проверка целостности (подпись опционально).
3. **Анализ LLM** – статический анализ кода (для Python) на вредоносные паттерны. Результат – предупреждение пользователю (не блокировка).
4. **Установка зависимостей** – если есть `requirements.txt`, создаётся venv и устанавливаются пакеты (разрешено только в песочнице плагина).
5. **Запрос разрешений** – при первом запуске плагин сообщает ядру, какие разрешения ему нужны (из `manifest.json`). Пользователь видит диалог с группировкой: “Обязательные” и “Опциональные”. Может выборочно разрешить/запретить.
6. **Выполнение** – плагин работает в отдельном потоке (Python) или в изолированном процессе (C++). При краше – не роняет клиент.
7. **Деактивация** – пользователь выключает плагин, ядро выгружает его и чистит ресурсы.

### 4.5. Безопасный режим
- Если приложение упало 1 раз при активном плагине – при следующем запуске **автоматически включается безопасный режим**: все плагины отключены.
- Пользователь видит уведомление: “Приложение упало из-за плагина X. Хотите перезапустить без него?”
- Можно посмотреть логи ошибки и после исправления перезапустить с плагином.

## 5. Разрешения (модель безопасности)

Каждое разрешение – строка, мапящаяся на конкретные действия.

**Категории разрешений** (пример, не исчерпывающий):

### Внутри Telegram
- `telegram.read_messages` – читать любые сообщения в любых чатах
- `telegram.send_messages` – отправлять сообщения от имени пользователя
- `telegram.read_contacts` – читать список контактов
- `telegram.modify_chats` – создавать/удалять чаты, менять настройки
- `telegram.call` – совершать звонки
- `telegram.read_stories` – просматривать сторис
- `telegram.delete_messages` – удалять сообщения (свои или чужие)

### Android / Система
- `android.camera` – доступ к камере
- `android.microphone` – доступ к микрофону
- `android.location` – геолокация
- `android.storage.read` – чтение внешнего хранилища (включая `/sdcard`)
- `android.storage.write` – запись на внешнее хранилище
- `android.storage.system` – доступ к `/data`, `/system` (требует SUDO)

### Привилегированные (SUDO)
- `sudo.root_command` – выполнение любой команды от root (включая запуск Tor, доступ к сетевым интерфейсам)
- `sudo.shizuku_api` – доступ к Shizuku API для привилегированных операций
- `sudo.network_capture` – перехват сетевого трафика (для анализа или проксирования)
- `sudo.install_packages` – установка/удаление приложений
- `sudo.gpu_access` – доступ к GPU для вычислений (нейросети)

### Специальные для Lumigram
- `lumigram.ghost_mode` – скрывать онлайны, статус печати, прочтения
- `lumigram.spy_mode` – сохранять удалённые сообщения, логировать звонки
- `lumigram.secret_storage` – читать/писать в защищённое хранилище (раздел 6)
- `lumigram.external_network` – разрешить плагину использовать собственные сетевые соединения (Tor, VPN, не через Telegram API)

**Правила выдачи**:
- Плагин не может сам себе дать разрешение.
- Пользователь может в любой момент отозвать любое разрешение в настройках плагина.
- Разрешения группируются по опасности (зелёные – безопасные, жёлтые – предупреждение, красные – root/привилегированные).
- Для SUDO-разрешений требуется отдельное подтверждение (пин-код или биометрия).

## 6. Защищённое хранилище (Secure Vault)

- Отдельное зашифрованное хранилище внутри Lumigram.
- Ключ шифрования привязан к паролю пользователя (или биометрии).
- Плагины могут запрашивать доступ к хранилищу через разрешение `lumigram.secret_storage`.
- Хранилище можно использовать для:
  - Скрытых чатов (блюр, пароль перед открытием).
  - Хранения конфиденциальных данных плагина (токены, ключи Tor, настройки нейросетей).
  - Резервных копий удалённых сообщений (шпионский режим).
- Реализация: SQLCipher или Android Keystore + AES-GCM.

## 7. Логирование и отладка

**Структурированный логгер**:
- Логи хранятся в SQLite (до 10000 записей, старые удаляются).
- Уровни: DEBUG, INFO, WARNING, ERROR, FATAL.
- Категории: PLUGIN_LOADER, PLUGIN_EXECUTION, PERMISSIONS, UI, NETWORK, SUDO и др.
- Каждая запись содержит: timestamp, уровень, категорию, ID плагина, сообщение, stack trace (если есть).

**UI логов** (в настройках Lumigram):
- Фильтры по уровню, категории, плагину.
- Поиск.
- Копирование сообщения об ошибке одной кнопкой.
- Экспорт логов в файл (JSON/TXT).

**При краше плагина**:
- Лог ошибки автоматически сохраняется.
- Показывается bulletin: “Plugin X crashed: java.lang.NullPointerException… [Копировать лог] [Отключить плагин]”.

## 8. SUDO / Shizuku интеграция

- **Shizuku** – позволяет приложению получать привилегии без root (через API). Используется для команд, не требующих полного root.
- **Root** – полный доступ через `su`.

**Пользовательский опыт**:
1. В настройках Lumigram есть переключатель “SUDO режим (экспериментально)”.
2. При первом включении – диалог: “Ты можешь дать плагинам больше прав, но это опасно. Продолжить?”.
3. Далее – управление для каждого плагина отдельно: в карточке плагина появляются SUDO-разрешения (запись в `/system`, выполнение команд, доступ к GPU и т.д.). По умолчанию все выключены.
4. Если плагин пытается выполнить действие без разрешения – лог и отказ.

**Технически**:
- C++ ядро имеет обёртку для `libsu` (shizuku) и `libroot`.
- Плагины вызывают `sudo_exec(cmd)`, ядро проверяет разрешение и выполняет через соответствующий бэкенд.

**Пример использования Tor в Python-плагине с root**:
```python
# Плагин запрашивает разрешения: sudo.root_command, lumigram.external_network
import subprocess
subprocess.run(["su", "-c", "tor -f /data/local/tmp/torrc &"])
# Далее плагин использует socks5 прокси localhost:9050 для обхода блокировок
```

## 9. Требования к отказоустойчивости

- Любое исключение в Python или C++ плагине **не должно приводить к вылету приложения**.
- Механизм: каждый плагин выполняется в отдельном потоке с перехватчиком исключений (`Thread.setUncaughtExceptionHandler` для Java-сторонних вызовов из Python). Для C++ – `std::set_terminate` + обработка сигналов (SIGSEGV, SIGABRT).
- Если плагин падает, система логирует и деактивирует плагин до следующего ручного включения.
- Максимальное время выполнения плагина на UI-потоке – предупреждение, если превышает 100 мс. Долгие задачи должны использовать `run_on_queue`.

## 11. Пример "шпионского" плагина с внешними библиотеками

```python
# spy_keeper.py
__id__ = "spy_keeper"
__name__ = "Spy Keeper"
__permissions__ = ["telegram.read_messages", "lumigram.spy_mode", "lumigram.secret_storage"]

# Импорт внешней библиотеки, установленной через requirements.txt (например, cryptography)
from cryptography.fernet import Fernet

class SpyPlugin(BasePlugin):
    def on_plugin_load(self):
        # Инициализация шифрования для хранилища
        key = self.secret_storage.get("encryption_key")
        if not key:
            key = Fernet.generate_key()
            self.secret_storage.set("encryption_key", key)
        self.cipher = Fernet(key)

    def on_update_hook(self, update_name, account, update):
        if update_name == "TL_updateDeleteMessages":
            for msg_id in update.messages:
                msg = get_message_by_id(msg_id)
                encrypted = self.cipher.encrypt(msg.serialize())
                self.secret_storage.save(f"deleted_{msg_id}", encrypted)
        return HookResult()
```

## 12. Технические ограничения

- Минимальная версия Android: 8.0 (API 26).
- Поддерживаемые архитектуры: arm64-v8a, armeabi-v7a.
- Python 3.11 с возможностью создания виртуальных окружений (`venv`).
- C++17 (для ядра).
- Запрещены: прямые вызовы `Runtime.exec()` без проверки через SUDO-менеджер, запись в системные разделы без явного разрешения, скачивание и выполнение произвольного кода (кроме как через установку `.plugin` из проверенного источника). Однако при наличии соответствующих разрешений (например, `sudo.root_command`) плагин может делать всё что угодно – ответственность на пользователе.
